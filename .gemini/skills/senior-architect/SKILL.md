# Senior Architect

## Diretrizes
- **Separation of Concerns**: Lógica de negócio (gates de fase) separada da camada de UI.
- **Patterns**: Use padrões como Repository ou Service Layer se a complexidade aumentar.
- **Scalability**: Design simples para começar, mas modular para evolução.
- **Marketing OS Rules**: Codifique as Regras de Bloqueio (Hard Gates) como lógica central.
