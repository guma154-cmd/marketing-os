# User Story 1.3: Template de Diagnóstico Base

**Como** profissional do Marketing OS,
**Quero** um template padronizado de diagnóstico-base,
**Para** transformar as respostas e dados coletados no briefing em uma leitura estratégica clara, identificando problemas, mapeando oportunidades e priorizando as ações operacionais.

## Descrição
Após a coleta de informações através do `briefing-base.md`, o próximo passo vital da metodologia Marketing OS é o cruzamento desses dados para gerar um **Diagnóstico** objetivo. Esta story prevê a criação do arquivo `docs/templates/diagnostico-base.md`. Este artefato servirá como ponte entre o que o cliente "disse" e o que a operação "precisa fazer", separando percepções superficiais de gargalos reais de negócio e preparando terreno para a proposta comercial.

## Critérios de Aceitação

### 1. Criação e Estrutura do Template
- [x] Criar o arquivo `docs/templates/diagnostico-base.md`.
- [x] Inserir bloco de instruções no topo ensinando como o profissional deve utilizar o arquivo e referenciar os dados do `briefing.md` correspondente.
- [x] Configurar a estrutura em Markdown para que seja facilmente replicável na pasta de cada cliente.

### 2. Leitura de Dados e Avaliação de Problemas/Oportunidades
- [x] O template deve conter uma seção de **Resumo do Cenário Atual** consumindo os dados vitais do briefing (Nicho, Oferta Principal, Canais de Aquisição Atuais).
- [x] Criar uma estrutura de "Sinais Vitais" (check-up) baseada no `marketing-os-base.md` (Ex: "O perfil transmite confiança?", "A oferta está clara?").
- [x] Organizar uma matriz ou lista categorizada para:
  - **Dores Visíveis:** O que o cliente relatou ou o que está óbvio no perfil.
  - **Dores Ocultas (Gargalos Reais):** O que a análise técnica identificou (ex: falta de CTA, tráfego levando para página quebrada).
  - **Oportunidades (Potencial):** Ativos subutilizados (ex: cliente tem ótimas provas sociais, mas não as publica).
  - **Riscos Operacionais:** Histórico ruim, desalinhamento de expectativa vs. verba.

### 3. Direcionamento e Priorização (Próximos Passos)
- [x] Adicionar seção para definir a **Prioridade de Ação** imediata (o que precisa ser resolvido nos primeiros 7, 15 e 30 dias).
- [x] Incluir área para **Recomendação de Produto** baseada no cruzamento de Orçamento x Necessidade, citando explicitamente os produtos: Presença Essencial, Presença + Captação ou Growth Local.
- [x] Garantir que o diagnóstico esteja estruturado de forma a facilitar a extração de dados para a elaboração de uma Proposta Comercial e do Plano de Ação.
