# User Story 2.1: Onboarding do Cliente

**Como** consultor do Marketing OS,
**Quero** um processo de onboarding estruturado e uma pasta para o primeiro cliente,
**Para** coletar materiais, acessos e contextos de forma organizada, garantindo que o preenchimento dos templates operacionais (Briefing/Diagnóstico) seja fluido e sem lacunas.

## Descrição
Esta story marca o início da **Operação Real**. Ela define como um novo cliente é "recebido" pelo sistema. O objetivo é criar o arquivo `onboarding.md` para o `cliente-01`, que servirá de checklist central para tudo o que o consultor precisa antes de começar a planejar o conteúdo.
