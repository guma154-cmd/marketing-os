# User Story 2.3: Diagnóstico Técnico do Cliente-01

**Como** consultor do Marketing OS,
**Quero** elaborar o diagnóstico técnico do cliente-01 utilizando o template base e os dados do briefing,
**Para** identificar o "gargalo real" do negócio, as oportunidades imediatas e recomendar o produto de serviço mais adequado com justificativa técnica.

## Descrição
O Diagnóstico é a etapa mais crítica da Fase 3. É aqui que o consultor exerce sua autoridade, lendo "nas entrelinhas" do briefing para detectar dores ocultas e riscos operacionais. O output desta story será o arquivo `diagnostico.md`, que guiará a execução dos primeiros 30 dias do cliente.

## Critérios de Aceitação

### 1. Estrutura e Coerência Técnica
- [x] Criar o arquivo `docs/clients/cliente-01/diagnostico.md`.
- [x] Seguir integralmente o template `docs/templates/diagnostico-base.md`.
- [x] Demonstrar rastro de dados ligando as conclusões do diagnóstico às respostas do briefing.

### 2. Análise de Dores e Oportunidades
- [x] Mapear **Dores Visíveis** (o que o cliente reclama) vs. **Dores Ocultas** (o que realmente trava o negócio).
- [x] Listar **Oportunidades** de "ganho rápido" (low hanging fruits).
- [x] Identificar **Riscos Operacionais** (ex: bloqueios de conta, falta de verba, atendimento ineficiente).

### 3. Prescrição e Planejamento
- [x] Definir a **Prioridade Imediata** (O que fazer no Dia 1?).
- [x] Estruturar o plano de ação inicial: **7, 15 e 30 dias**.
- [x] Apresentar a **Recomendação de Produto** (Essencial, Captação ou Growth) com justificativa estratégica baseada nos sinais coletados.
- [x] Listar pendências de dados que ainda possam enviesar a precisão desta análise.
